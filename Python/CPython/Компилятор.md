еПосле того, как был исходный код прошел парсинг (имеется дерево AST - операции, классы, функции, пространства имен) его необходимо преобразовать в инструкции понятные для процессора (Python виртуальной машины).

```
(Модуль чтения) --текст--> 
(Модуль лексического анализа) --CST-->
(Парсер) --AST-->
(Компилятор) --CFG-->  👈🏻
(Ассемблер) --Байткод-->  👈🏻
(Выполнение)
```
В компиляции участвуют два компонента:
- **Компилятор**. Преобразование AST к графу потока управления CFG (Control Flow Graf) представляющий логическую последовательность выполнения (какое отличие от AST?)
- **Ассемблер**. Преобразование CFG к инструкциям байт-кода

Входная точка компиляции функция `PyAST_CompileObject` (`Python/compile.c`)

**Новые термины**
- *Состояние компилятора*. Реализуется как тип контейнера (что за контейнер?), содержащий одну *таблицу символических имен*.
- *Таблица символических имен* содержит имена переменных. Также, в ней могу находится дочерние таблицы символических имен.
- *Состояние компилятора* включает в себя несколько *единиц компиляции*.
- Каждая *единица компиляции* может содержать множество *имен*, *имен переменных*, *констант* (которые в Си?) и *сегментных переменных* (имеется ввиду выделенные переменные в куче через Си?).
- *Единица компиляции* содержит несколько *базовых блоков*.
- *Базовые блоки* содержат множество *инструкций байт кода*.
